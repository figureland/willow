0	Upload	User uploads a Sandy template, or drops any file(s) for AI extraction (Excel, CSV, PDF, FMS exports). Multiple files accepted simultaneously. System detects file type and routes accordingly.
1	Summary review	Sandy presents a data receipt — total records found, years covered, data categories identified, confidence level per category. User confirms scope before proceeding. Nothing is saved yet.
2	Mapping	Sandy maps values from the file to its internal catalogue (crop names, product names, fuel types, units). Unmatched items are surfaced for user review. Bulk mapping applies one correction to all matching records. Mappings are saved for future uploads from the same source.
3	Validation	Rule-based checks run across all records. Issues are shown in three tiers: blocking (must fix to proceed), warnings (affect accuracy), notes (Sandy has made an assumption). Inline fixes — user never leaves the screen to resolve an issue.
4	Anomaly detection	Statistical outlier review. Distribution charts show farm values versus typical range for region and enterprise type. User confirms, corrects, or excludes flagged values. All values default to confirmed — Sandy flags for awareness, not to make decisions.
5	Completeness	Gap analysis showing what's missing, what it unlocks, and whether Sandy can prefill. Sandy prefill shows exactly what it would add and from which source before applying. User can mix: prefill some gaps, skip others, add a few manually.
6	Commit	Pre-commit summary shows exactly what will be saved — confirmed records, prefilled records, and skipped items. Single Save to Sandy action. Nothing in the live farm record changes until this step.
